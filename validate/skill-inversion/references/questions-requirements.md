# 需求确认问题模板

## 核心问题（必问）

1. **这个功能要解决什么用户痛点？** 用户现在怎么做的？为什么不满意？
2. **核心用户流程是什么？** 用户从打开到完成的主要步骤是什么？
3. **有没有必须遵守的业务规则？** 比如权限控制、数据校验、合规要求。
4. **验收标准是什么？** 怎样算"做完了"？有具体的测试场景吗？
5. **优先级如何排？** 如果必须砍功能，哪些可以延后？

## 补充问题（按需追问）

6. 有没有 UI 设计稿或原型图？
7. 需要支持哪些平台/设备？（Web / 移动 / 桌面）
8. 数据从哪里来？需要对接哪些外部系统？
9. 有没有性能要求？（响应时间、并发量）
10. 需要支持国际化/多语言吗？

## 跳过条件

- 如果是 API 开发（无 UI） → 跳过问题 6
- 如果用户给了完整的需求文档 → 聚焦问题 4、5 做确认
- 如果是内部工具 → 简化问题 7（通常只需 Web）
