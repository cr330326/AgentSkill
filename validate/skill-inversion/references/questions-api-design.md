# 接口/数据模型设计问题模板

## 核心问题（必问）

1. **API 的消费者是谁？** 前端页面 / 移动 App / 第三方系统 / 内部微服务？
2. **协议偏好？** REST / GraphQL / gRPC / WebSocket？有没有已有的规范要遵循？
3. **核心实体有哪些？** 列举主要的数据对象及其关系（1:N、N:N）。
4. **读写比例大概多少？** 读多写少？还是写入密集？（影响数据模型设计）
5. **认证和授权方案？** JWT / OAuth / API Key？需要哪些权限粒度？

## 补充问题（按需追问）

6. 需要版本管理吗？（API 版本化策略：URL / Header / Query）
7. 分页方案？Offset / Cursor？预期数据量级？
8. 错误码规范？用 HTTP 状态码还是自定义业务码？
9. 是否需要幂等性保证？（支付、订单等场景）
10. 数据校验放在哪层？（客户端 / 网关 / 服务端 / 数据库）

## 跳过条件

- 如果是内部微服务间通信 → 简化问题 5（通常用服务网格认证）
- 如果用户已给了 OpenAPI 规范 → 跳过大部分，聚焦审查和补充
- 如果是 CRUD 简单场景 → 问题 4、6-10 可简化
