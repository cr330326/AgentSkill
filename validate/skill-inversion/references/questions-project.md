# 项目规划问题模板

## 核心问题（必问）

1. **项目目标是什么？** 用一句话描述这个项目要解决什么问题。
2. **目标用户是谁？** 内部团队 / 外部客户 / 两者都有？
3. **技术栈有限制吗？** 是否必须使用某些语言、框架、平台？
4. **时间线是什么？** 有明确的截止日期吗？大概多长时间完成？
5. **团队情况如何？** 几个人开发？技术背景如何？

## 补充问题（按需追问）

6. 是否有现有代码库要兼容？还是从零开始？
7. 部署环境是什么？（本地服务器 / 云 / 容器 / Serverless）
8. 有没有类似的产品可以参考？（竞品或内部已有系统）
9. 预算有限制吗？（影响技术选型，比如能否用付费服务）
10. MVP 需要哪些核心功能？可以先砍掉哪些？

## 跳过条件

- 如果用户已经给了技术栈 → 跳过问题 3
- 如果是个人项目 → 跳过问题 5
- 如果用户说"快速原型" → 聚焦问题 1、3、10，其余简化
