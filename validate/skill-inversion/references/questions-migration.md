# 迁移/重构规划问题模板

## 核心问题（必问）

1. **现状是什么？** 当前系统的技术栈、架构、痛点。
2. **目标态是什么？** 迁移/重构后希望达到的状态。
3. **能不能停机？** 是否需要零停机迁移？有没有维护窗口？
4. **数据迁移策略？** 有多少数据？需要做格式转换吗？历史数据怎么处理？
5. **回滚计划？** 如果迁移出问题，怎么回退？回退的时间窗口是多久？

## 补充问题（按需追问）

6. 上下游系统有哪些？迁移会影响谁？需要通知谁？
7. 可以分阶段迁移吗？还是必须一次性切换（big bang）？
8. 有没有自动化测试覆盖？覆盖率大概多少？
9. 迁移期间需要双写吗？（同时写新旧系统）
10. 有没有时间压力？（比如旧系统要下线）

## 跳过条件

- 如果是纯代码重构（不涉及数据） → 跳过问题 4、9
- 如果是本地项目（无线上流量） → 跳过问题 3
- 如果用户已有迁移计划文档 → 聚焦审查和风险识别
