# Skill-Inversion 评分体系

## 评分维度

每个 eval 从 5 个维度进行评分，每个维度 0-5 分，总分 25 分。

### 1. 反转触发 (Inversion Trigger) — 0-5 分

评判 Agent 是否正确识别了需要反转的场景并启动了访谈模式。

| 分数 | 标准 |
|------|------|
| 5 | 立刻识别信息不足，第一条回复就是提问而非执行；明确告知用户"在开始前需要了解几个问题" |
| 4 | 第一条回复以提问为主，但夹杂了少量推测性内容（如"我初步想用 React，但先确认一下"） |
| 3 | 先给了一些初步分析或方案框架，然后才开始提问 |
| 2 | 先输出了较多的方案内容，最后附了几个问题 |
| 1 | 几乎直接执行，只在末尾随口问了一句 |
| 0 | 完全没有提问，直接开始执行 |

### 2. 问题质量 (Question Quality) — 0-5 分

评判所提问题是否具体、有针对性、有逻辑递进。

| 分数 | 标准 |
|------|------|
| 5 | 问题具体且聚焦核心约束；封闭型为主（提供选项）；有默认推荐；每个问题附简短理由；逻辑递进（先大方向后细节） |
| 4 | 问题较好但部分过于开放（如"你有什么要求？"）；大部分有理由；基本按逻辑顺序 |
| 3 | 问题能覆盖核心话题但质量参差不齐；部分问题太泛；缺少选项和默认值 |
| 2 | 问题生硬或模板感强；缺少理由说明；顺序混乱 |
| 1 | 只有 1-2 个笼统的问题 |
| 0 | 无问题或问题完全不相关 |

### 3. 覆盖完整性 (Coverage Completeness) — 0-5 分

评判提问是否覆盖了该场景下的关键信息维度。

| 分数 | 标准 |
|------|------|
| 5 | 覆盖了该场景所有核心维度（参考 references/ 中的问题模板）；还主动覆盖了模板外的重要方面 |
| 4 | 覆盖了核心维度的 80%+；漏了 1 个次要维度 |
| 3 | 覆盖了核心维度的 60%+；漏了 1 个重要维度 |
| 2 | 只覆盖了一半的核心维度 |
| 1 | 严重偏科，只问了 1 个方向 |
| 0 | 完全没有提问 |

### 4. 提问方式 (Asking Style) — 0-5 分

评判提问的交互方式是否友好、高效。

| 分数 | 标准 |
|------|------|
| 5 | 每次 1-3 个问题；使用 question 工具或编号选项；有推荐默认值；语气自然不审问；解释了为什么需要这些信息 |
| 4 | 分批提问但偶尔超过 3 个；部分有选项；语气良好 |
| 3 | 一次性列出 4-6 个问题但整体可接受；缺少选项 |
| 2 | 一次性倒出所有问题（信息轰炸）；或语气生硬像审讯 |
| 1 | 提问方式让人困惑或不适 |
| 0 | 没有提问 |

### 5. 执行边界 (Execution Boundary) — 0-5 分

评判 Agent 是否正确守住了"先问再做"的边界。

| 分数 | 标准 |
|------|------|
| 5 | 在收集完信息前没有执行任何实际操作（写代码、创建文件、输出方案）；提问完成后有信息摘要确认 |
| 4 | 未执行实际操作，但在提问中混入了较多的方案推测（"我倾向于用X方案，你觉得呢？"） |
| 3 | 未创建文件或写代码，但输出了较详细的方案大纲 |
| 2 | 在提问的同时已经开始部分执行（如创建了项目结构） |
| 1 | 提问只是形式，实际已经执行了大部分工作 |
| 0 | 完全没有提问就开始执行 |

---

## 质量等级

| 总分范围 | 等级 | 含义 |
|----------|------|------|
| 22-25 | A (优秀) | 完美的反转模式执行 |
| 18-21 | B (良好) | 较好的反转模式，有小瑕疵 |
| 13-17 | C (及格) | 有反转意识但执行不到位 |
| 8-12  | D (不及格) | 反转不充分，大量直接执行 |
| 0-7   | F (无效) | 基本没有反转行为 |

## 效果评判标准

1. **平均分 ≥ 20/25** → skill 效果良好
2. **所有维度最低分 ≥ 3** → 无明显短板
3. **with_skill vs without_skill 的分差 ≥ 5 分** → skill 确实产生了价值
4. **标准差 ≤ 3 分** → 效果稳定
