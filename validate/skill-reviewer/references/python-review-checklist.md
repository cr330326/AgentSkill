## 命名与风格

### PY-01 命名规范
- 变量和函数使用 `snake_case`
- 类使用 `PascalCase`
- 常量使用 `UPPER_SNAKE_CASE`
- 私有成员以 `_` 前缀
- 避免单字母变量名（循环索引 `i`, `j`, `k` 除外）

### PY-02 导入顺序
按照 PEP 8 规定的顺序分组：
1. 标准库
2. 第三方库
3. 本地模块

各组之间用空行分隔。

### PY-03 行长度
单行不超过 120 字符（团队约定可放宽到 120，但不应超过）。

### PY-04 空行与缩进
- 顶层函数/类之间 2 个空行
- 类内方法之间 1 个空行
- 使用 4 空格缩进，不混用 tab

### PY-05 无用代码
- 不应有被注释掉的代码块（用版本控制代替）
- 不应有未使用的 import
- 不应有未使用的变量

## 类型标注与文档

### PY-06 函数签名类型标注
公开函数的参数和返回值应有类型标注。

### PY-07 复杂类型使用 typing 模块
`Optional`, `Union`, `List`, `Dict` 等复杂类型应使用 `typing` 模块或 Python 3.10+ 的内置语法 (`X | Y`)。

### PY-08 Docstring
公开函数和类应有 docstring，说明：
- 功能描述
- 参数说明（类型不明显时）
- 返回值说明
- 可能抛出的异常

### PY-09 行内注释
- 注释应解释"为什么"而非"是什么"
- 复杂逻辑处应有注释
- 注释应与代码保持同步

### PY-10 TODO/FIXME
TODO 和 FIXME 注释应包含责任人和日期，或关联到 issue。

## 异常处理

### PY-11 禁止裸 except
不要使用 `except:` 或 `except Exception:`，应捕获具体的异常类型。

### PY-12 异常信息
捕获的异常应记录有意义的错误信息，包含上下文（如哪个操作失败、涉及什么数据）。

### PY-13 不要吞掉异常
`except` 块中不应只有 `pass`（除非有充分理由并注释说明）。

### PY-14 使用 raise from
重新抛出异常时使用 `raise NewError(...) from original_error` 保留异常链。

### PY-15 finally 与资源清理
文件、连接等资源应使用 `with` 语句或 `try/finally` 确保清理。

### PY-16 自定义异常
业务逻辑应定义自定义异常类，而非复用通用异常。

## 代码结构与复杂度

### PY-17 函数长度
单个函数不超过 50 行（不含空行和注释）。超过时考虑拆分。

### PY-18 函数参数数量
函数参数不超过 5 个。超过时考虑使用 dataclass 或 dict 封装。

### PY-19 嵌套深度
条件/循环嵌套不超过 3 层。使用 early return、guard clause 降低嵌套。

### PY-20 单一职责
每个函数只做一件事。如果函数名中有"和"字，可能需要拆分。

### PY-21 重复代码
相似逻辑出现 3 次及以上时应提取为公共函数。

### PY-22 魔法数字
代码中不应有未解释的数字常量（0、1、空字符串等除外），应提取为命名常量。

## 安全与健壮性

### PY-23 输入验证
外部输入（用户输入、API 参数、文件内容）在使用前应验证类型和范围。

### PY-24 SQL 注入
数据库查询使用参数化查询，禁止字符串拼接 SQL。

### PY-25 路径遍历
文件操作应验证路径不包含 `..`，使用 `pathlib` 处理路径拼接。

### PY-26 敏感信息
密码、API Key、Token 不应硬编码在源码中，应使用环境变量或配置文件。

### PY-27 eval/exec
禁止使用 `eval()` 和 `exec()` 处理用户输入。

### PY-28 pickle
`pickle.load()` 不应用于加载不可信来源的数据。

## 性能

### PY-29 循环中的重复计算
循环体内不应有可以提到循环外的计算（如 `len()` 调用、正则编译、数据库查询）。

### PY-30 数据结构选择
- 频繁查找使用 `set` 或 `dict` 而非 `list`
- 大量字符串拼接使用 `join()` 而非 `+`

### PY-31 生成器
处理大量数据时优先使用生成器（`yield`）而非一次性加载到列表。

### PY-32 N+1 查询
ORM 使用时检查是否存在 N+1 查询问题。

### PY-33 缓存
重复调用的纯函数考虑使用 `functools.lru_cache` 缓存结果。

## 测试相关

### PY-34 测试覆盖
公开 API 的核心路径应有测试覆盖。

### PY-35 测试命名
测试函数名应描述被测试的行为：`test_login_with_invalid_password_returns_401`。

### PY-36 测试隔离
测试之间不应有状态依赖，每个测试独立运行。

### PY-37 Mock 使用
只 mock 外部依赖（网络、文件系统、数据库），不 mock 被测模块内部逻辑。
