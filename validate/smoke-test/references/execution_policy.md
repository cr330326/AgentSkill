# Execution Policy

只有在满足下面条件时，才允许实际执行 smoke test。

## 必须满足

1. 用户明确要求执行，而不是只生成。
2. 用户提供了 `service_url`。
3. 用户明确确认该地址可被访问。
4. 环境中可用 `pytest`。

## 默认禁止的行为

- 未确认就请求线上或生产类地址
- 自动安装 `pytest`、`pyyaml` 或其他依赖
- 自动修改项目正式测试目录或 CI 配置
- 自动为鉴权接口伪造 token、cookie、签名或业务参数

## 建议的确认问题

执行前至少确认以下信息：

- 这是 staging、preprod 还是 production 地址？
- 是否允许对该地址发起只读健康检查请求？
- 如果 OpenAPI 中只有写操作接口，是否还要继续生成但不执行？

## 执行降级规则

- 缺少 `pytest`: 只生成，不执行。
- 缺少 `service_url`: 只生成，不执行。
- 候选端点全部需要鉴权或写操作: 只生成，并把原因写入结果。

## 结果要求

无论是否执行，都要返回结构化 JSON，并明确标记：

- 是否执行过
- 为什么没有执行
- 执行用了哪个 `service_url`
- 哪些端点被跳过以及原因
