# Skill Generator 评分体系

## 评分维度

生成的 skill 从 5 个维度进行评分，每个维度 0-5 分，总分 25 分。

### 1. 结构合规性 (Structure Compliance) — 0-5 分

评判生成的 skill 是否符合 SKILL.md 规范。

| 分数 | 标准 |
|------|------|
| 5 | frontmatter 完美（name kebab-case + description 50-150词 + 含触发/反触发）；目录结构干净；无空目录；引用文件全部存在 |
| 4 | frontmatter 正确但 description 略短/略长；或缺少反触发条件；结构基本合规 |
| 3 | frontmatter 基本正确但有小问题（如 description 太短 <30词）；有一处引用文件缺失 |
| 2 | frontmatter 有语法问题（但可解析）；或缺 name/description 之一；多处引用缺失 |
| 1 | frontmatter 不可解析；或完全缺失 frontmatter |
| 0 | 没有生成 SKILL.md |

**自动化检测项：**
- `quick_validate.py` 通过 → 基础分 2
- name 是 kebab-case → +0.5
- description 长度 50-150 词 → +0.5
- description 含 "Use when" / "Use this skill when" → +0.5
- description 含 "Do NOT use" / 反触发 → +0.5
- 无空子目录 → +0.5
- 所有引用文件存在 → +0.5

### 2. 触发质量 (Trigger Quality) — 0-5 分

评判 description 能否在正确的场景触发、错误的场景不触发。

| 分数 | 标准 |
|------|------|
| 5 | description 覆盖多种表达方式（中英文、口语化、正式化）；含具体文件类型/CLI名；有清晰的反触发；与相邻 skill 有区分 |
| 4 | 触发短语较全面但缺少某类表达（如只有英文没中文）；有反触发 |
| 3 | 能覆盖核心场景但遗漏边缘触发；无反触发 |
| 2 | 只有一两个触发词，过于泛化或过于窄 |
| 1 | description 是纯功能描述，无触发条件 |
| 0 | 无 description |

### 3. 指令可执行性 (Instruction Actionability) — 0-5 分

评判 body 中的指令是否具体、可操作、能被 Claude 直接执行。

| 分数 | 标准 |
|------|------|
| 5 | 每个步骤都是具体动作（祈使语气）；包含命令/代码示例；有决策表辅助选择；解释了 why |
| 4 | 步骤具体但少量地方偏描述性；示例充足；有 why 解释 |
| 3 | 有明确的步骤但部分过于笼统（如"适当处理"）；示例不足 |
| 2 | 多数指令是描述性的（"该 skill 会做 X"而不是"做 X"）；缺少示例 |
| 1 | 全部是功能描述，无可执行指令 |
| 0 | body 为空或无意义 |

**关键指标：**
- 使用祈使语气的句子占比 ≥ 60% → 高分
- 包含 ≥ 2 个代码/命令示例 → +1
- 包含决策表 → +0.5
- 包含 before/after 对比示例 → +0.5

### 4. 内容完整性 (Content Completeness) — 0-5 分

评判 skill 是否完整覆盖了用户请求的功能。

| 分数 | 标准 |
|------|------|
| 5 | 完全覆盖用户所有需求点；有额外的边缘情况处理；troubleshooting 部分 |
| 4 | 覆盖所有核心需求；缺少 1-2 个边缘场景 |
| 3 | 覆盖大部分需求但遗漏了一个重要功能 |
| 2 | 只覆盖了约一半的需求 |
| 1 | 严重偏离用户需求 |
| 0 | 与请求无关 |

### 5. 模式匹配度 (Pattern Fitness) — 0-5 分

评判是否选择了合适的 skill 模式，体量是否合理。

| 分数 | 标准 |
|------|------|
| 5 | 模式选择完美匹配用例；body 长度在模式推荐范围内；层次结构清晰；progressive disclosure 合理 |
| 4 | 模式基本合适；长度略有偏差但不影响使用 |
| 3 | 模式可接受但不是最优选择（如简单工具用了 practice guide 的重量级结构） |
| 2 | 模式明显不匹配（如复杂流程只写了 60 行） |
| 1 | 无明显模式，结构混乱 |
| 0 | 无结构 |

## 质量等级

| 总分范围 | 等级 | 含义 |
|----------|------|------|
| 22-25 | A (优秀) | 生产就绪，可直接使用 |
| 18-21 | B (良好) | 小幅修改即可使用 |
| 13-17 | C (及格) | 需要较多修改但框架可用 |
| 8-12  | D (不及格) | 需要大幅重写 |
| 0-7   | F (无效) | 基本不可用 |

## 效果评判标准

skill-generator 的效果通过以下指标综合评判：

1. **平均分 ≥ 20/25** → skill 效果良好
2. **所有维度最低分 ≥ 3** → 无明显短板
3. **with_skill vs without_skill 的分差 ≥ 5 分** → skill 确实产生了价值
4. **3 个测试用例的标准差 ≤ 3 分** → 效果稳定，不是靠运气
