# 数据处理流水线

## 适用场景

批量数据处理时使用：数据清洗、格式转换、数据迁移、ETL（提取-转换-加载）、
批量文件处理等。数据处理通常涉及大量记录，中间某步出错可能导致数据污染，
流水线确保每个环节的数据质量都经过确认。

## 阶段定义

### 阶段 1: 数据源分析 [CHECKPOINT]

- **目标**: 理解输入数据的结构、质量和规模
- **输入**: 用户指定的数据源（文件、数据库、API 等）
- **操作**:
  1. 读取数据源基本信息（文件大小、记录数、字段列表）
  2. 数据质量概览：
     - 各字段的空值率
     - 数据类型分布
     - 明显的异常值或格式问题
  3. 抽样展示前 5-10 条记录
  4. 识别潜在的数据问题（编码问题、分隔符不一致、日期格式混乱等）
- **输出**: 数据源分析报告（字段列表、数据质量概览、问题清单）
- **检查要点**: 数据源是否符合预期？识别出的问题是否需要在清洗阶段处理？

### 阶段 2: 处理方案设计 [CHECKPOINT]

- **目标**: 设计数据处理的具体规则
- **输入**: 阶段 1 的分析报告 + 用户的处理需求
- **操作**:
  1. 定义每个字段的清洗规则（去空格、统一格式、填充默认值等）
  2. 定义转换规则（字段映射、计算字段、数据聚合等）
  3. 定义过滤/筛选条件
  4. 定义输出格式和目标
  5. 编写处理规则清单（每条规则一个编号，如 RULE-01）
- **输出**: 处理方案文档（规则清单 + 字段映射表）
- **检查要点**: 处理规则是否完整？是否有遗漏的边界情况？

### 阶段 3: 试运行 [CHECKPOINT]

- **目标**: 用小样本验证处理方案的正确性
- **输入**: 阶段 2 的处理方案 + 原始数据的子集
- **操作**:
  1. 从原始数据中抽取样本（前 100 条 或 随机 1%）
  2. 按处理方案执行全部规则
  3. 展示处理前后的对比（至少 5 条）
  4. 统计：
     - 处理成功率
     - 被过滤掉的记录数
     - 转换后各字段的值分布
  5. 记录执行耗时，估算全量处理时间
- **输出**: 试运行报告（前后对比 + 统计数据 + 耗时估算）
- **检查要点**: 处理结果是否符合预期？有无意外的数据丢失或变形？

### 阶段 4: 全量执行

- **目标**: 对全部数据执行处理
- **输入**: 阶段 3 确认通过的处理方案 + 全量原始数据
- **操作**:
  1. 备份原始数据（如果是文件操作，复制一份；如果是数据库，记录快照点）
  2. 按处理方案执行全部规则
  3. 记录处理日志（每批次的处理条数、错误数）
  4. 对于大数据量，分批处理并显示进度
- **输出**: 处理后的数据 + 执行日志

### 阶段 5: 结果验证 [CHECKPOINT]

- **目标**: 验证全量处理结果的正确性
- **输入**: 阶段 4 的处理结果
- **操作**:
  1. 基础校验：
     - 记录总数对比（原始 vs 处理后，差异是否可解释）
     - 关键字段的值分布对比
     - 抽样检查（随机 10 条，人工核对）
  2. 业务规则校验：
     - 必填字段是否有空值
     - 唯一键是否有重复
     - 数值范围是否合理
  3. 与阶段 3 试运行结果对比，确认一致性
- **输出**: 验证报告（校验项通过/失败列表 + 抽样结果）
- **检查要点**: 验证是否全部通过？失败项的影响范围和严重程度？

### 阶段 6: 输出交付

- **目标**: 将处理结果交付到目标位置
- **输入**: 阶段 5 确认通过的处理结果
- **操作**:
  1. 按目标格式导出（CSV / JSON / Excel / 数据库写入等）
  2. 如果需要，生成数据字典/元数据说明
  3. 输出处理汇总报告：
     - 原始记录数 → 最终记录数
     - 清洗掉的异常数据数量
     - 各项转换规则的执行统计
- **输出**: 最终数据文件 + 处理汇总报告

## 常用数据处理操作速查

| 操作类型 | 典型操作 | 注意事项 |
|---------|---------|---------|
| 清洗 | 去空格、统一大小写、修复编码、去重 | 去重要定义"重复"的判断字段 |
| 转换 | 日期格式化、单位换算、字段拆分/合并 | 保留原始字段还是覆盖？ |
| 过滤 | 删除空行、按条件筛选 | 被过滤的数据是否需要单独保存？ |
| 聚合 | 分组统计、求和/均值/计数 | 聚合粒度和维度需提前确认 |
| 映射 | 字段重命名、值映射（A→1, B→2） | 未匹配到的值如何处理？ |

## 注意事项

- **始终备份**：全量处理前必须备份原始数据，这是底线
- **编码陷阱**：CSV 文件优先用 UTF-8；Excel 导出的 CSV 可能是 GBK/Shift-JIS，处理前先确认
- **大文件策略**：超过 100MB 的文件建议分块读取处理（如 pandas 的 `chunksize`），避免内存溢出
- **幂等性**：处理脚本应设计为可重复执行——运行两次的结果应该和运行一次相同
