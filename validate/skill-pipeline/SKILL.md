# pipeline Skill
## 用途
对于复杂任务，强制执行严格的顺序工作流，在关键节点设置硬性检查点，需用户确认后才能进入下一步，避免跳过步骤导致错误。
适合场景：
- 文档生成流水线
- 发布流程
- 多步骤复杂任务
- 数据处理流水线
## 结构
```
pipeline/
├── SKILL.md
└── references/
    └── doc-pipeline.md # 文档流水线步骤定义
    └── release-pipeline.md # 发布流水线步骤定义
```
## 使用规则
1. 接到对应任务时，首先加载`references/`下对应场景的流水线定义
2. 严格按步骤顺序执行，禁止跳过任何步骤
3. 每完成一个关键节点，输出当前阶段结果，等待用户确认
4. 用户确认通过后才能进入下一步
5. 所有步骤完成后输出最终结果
## 激活条件
当用户输入包含以下关键词时自动激活：
- 文档流水线, 发布流程, 多步骤任务, 数据处理流水线
## 输出要求
必须严格按步骤执行，未得到用户确认禁止进入下一步。
