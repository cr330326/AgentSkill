## [阶段 1/6] 发布前检查 ✅ 完成

以下是针对您的 npm 包的发布前检查清单：

### 1. 分支状态
- [ ] 当前处于主发布分支（`main` 或 `master`）
- [ ] 该分支已与远程同步（`git pull` 无新提交）
- [ ] 没有未合并的 PR 阻塞本次发布

### 2. 工作区干净度
- [ ] `git status` 无未提交的变更（工作区干净）
- [ ] 没有遗留的临时文件或调试代码

### 3. 测试套件
- [ ] `npm test` 全部通过，无失败用例
- [ ] 新增的两个功能有对应的测试覆盖
- [ ] bug 修复有对应的回归测试

### 4. Lint / 类型检查
- [ ] `npm run lint`（或等效命令）无错误、无警告
- [ ] 若项目使用 TypeScript，`tsc --noEmit` 无类型错误

### 5. CI/CD 状态
- [ ] 最新 commit 的 CI 流水线（GitHub Actions / 其他）全部绿灯
- [ ] 无已知的 CI 配置问题

### 6. 依赖检查
- [ ] `npm audit` 无高危或严重漏洞
- [ ] `package.json` 中的依赖版本合理，无意外锁定

### 7. 版本号初步判断
根据 semver 规则分析本次变更：
- 2 个**新功能** → 需要 minor 升级
- 1 个 **bug 修复** → patch 级别（被 minor 包含）
- 无破坏性变更 → 不需要 major 升级

**建议版本号：`1.2.3` → `1.3.0`**

版本号将在阶段 2 中正式确定，届时同步更新 `package.json` 的 `version` 字段并生成 CHANGELOG。
