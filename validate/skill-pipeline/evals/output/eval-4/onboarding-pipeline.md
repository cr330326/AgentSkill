# 客户上线流水线

## 适用场景
当有新客户签约后，需要从合同签署到正式上线的完整交付流程时使用此流水线。适用于 SaaS 产品、企业级软件等需要逐步开通和交付的场景。每个客户上线都应走完此流水线的全部阶段，确保交付质量和客户体验。

## 阶段定义

### 阶段 1: 签署合同 [CHECKPOINT]
- **目标**: 完成合同的签署与归档，确认客户信息和服务范围
- **输入**: 商务谈判结果、合同模板、客户基本信息
- **操作**:
  1. 根据商务条款生成正式合同
  2. 双方签署合同（线上或线下）
  3. 合同归档并录入系统
  4. 确认服务范围、有效期、SLA 等关键条款
- **输出**: 已签署的合同文件、客户信息表、服务范围说明
- **检查要点**:
  - 合同是否经双方签署并盖章
  - 服务范围、价格、有效期是否与商务沟通一致
  - 客户联系人和技术对接人信息是否完整
  - 付款条件和发票信息是否确认

### 阶段 2: 创建客户账号
- **目标**: 在系统中创建客户的组织和管理员账号
- **输入**: 客户信息表、服务范围说明
- **操作**:
  1. 在系统中创建客户组织（租户）
  2. 创建客户管理员账号
  3. 发送账号激活邮件
  4. 确认客户管理员能够成功登录
- **输出**: 客户组织 ID、管理员账号凭证、登录确认记录

### 阶段 3: 配置权限
- **目标**: 根据合同约定的服务范围配置功能权限和角色
- **输入**: 服务范围说明、客户组织 ID
- **操作**:
  1. 根据服务套餐开通对应的功能模块
  2. 配置角色和权限模板
  3. 设置数据隔离和访问边界
  4. 验证权限配置是否生效
- **输出**: 权限配置清单、功能开通确认

### 阶段 4: 导入历史数据 [CHECKPOINT]
- **目标**: 将客户的历史业务数据安全、完整地迁移到新系统
- **输入**: 客户提供的历史数据文件、数据映射规范、客户组织 ID
- **操作**:
  1. 与客户确认数据格式和字段映射关系
  2. 对数据进行清洗和格式转换
  3. 在测试环境执行试导入并验证
  4. 在生产环境执行正式导入
  5. 生成数据导入报告（成功/失败/跳过条数）
- **输出**: 数据导入报告、数据校验结果
- **检查要点**:
  - 导入前是否已在测试环境验证通过
  - 数据条数和关键字段是否与源数据一致
  - 是否存在导入失败或跳过的记录，原因是否已确认
  - 客户是否确认抽检数据准确性

### 阶段 5: 培训
- **目标**: 确保客户团队掌握系统的核心操作和日常使用方法
- **输入**: 客户联系人信息、培训计划模板、系统操作手册
- **操作**:
  1. 与客户确认参训人员和培训时间
  2. 根据客户角色定制培训内容
  3. 执行线上或线下培训
  4. 培训后进行问答和操作演练
  5. 发放培训资料和操作手册
- **输出**: 培训签到记录、培训反馈表、操作手册交付确认

### 阶段 6: 正式上线 [CHECKPOINT]
- **目标**: 客户系统正式切换为生产使用状态
- **输入**: 前五个阶段全部完成的确认记录、上线检查清单
- **操作**:
  1. 执行上线前检查清单（账号、权限、数据、培训全部就绪）
  2. 与客户确认上线日期和切换方案
  3. 切换系统状态为正式运行
  4. 监控上线后首日运行状况
  5. 进入为期 1-2 周的重点保障期
- **输出**: 上线确认书、保障期支持计划、客户交付报告
- **检查要点**:
  - 前五个阶段是否全部完成且无遗留问题
  - 客户是否已签署上线确认书
  - 回滚方案是否已准备就绪
  - 保障期的支持团队和响应机制是否已到位

## 注意事项
- 阶段 1（合同签署）完成后才可启动后续阶段，这是整条流水线的前提
- 阶段 4（数据导入）涉及外部数据源操作，必须先在测试环境验证，避免生产数据污染
- 阶段 6（正式上线）为不可逆操作，上线前必须确保回滚方案可用
- 各阶段之间建议保留 1-2 个工作日的缓冲时间
- 全流程建议指定一名项目经理作为主负责人，确保阶段间交接顺畅
- 每个 CHECKPOINT 阶段需要相关负责人明确确认后才可进入下一阶段
